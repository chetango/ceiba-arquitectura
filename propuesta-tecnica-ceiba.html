El Problema

Los países de América Latina enfrentan un volumen masivo de hurto vehicular. Las cámaras de monitoreo existen, pero no están conectadas con las bases de datos policiales — cada país opera en un silo.

Problema técnico central: conectar hardware heterogéneo en campo, microservicios en la nube y bases de datos policiales de múltiples países — con cero tolerancia a pérdida de eventos.

La Solución: 3 Capas

📷 Capa Edge — Dispositivos en Calle
Worker Service .NET en cada cámara. Patrón store-and-forward con SQLite — ningún evento se pierde aunque caiga la conectividad o la energía. Comunicación vía MQTT con mTLS.
☁️ Capa Cloud — Microservicios en Kubernetes
5 microservicios ASP.NET Core / Worker Service: Ingesta, Detección, Consulta, Notificaciones y Analytics. Desacoplados por RabbitMQ. Los mismos contenedores corren en AWS, Azure o GCP sin cambiar una línea.
🔗 Capa de Integración — Heterogeneidad por País
Patrón Adapter por país para normalizar bases de datos de vehículos hurtados. Keycloak como broker de identidades que federa LDAP, AD, SOAP y REST sin tocar el sistema central.

📦 Store-and-Forward

  • Siempre guarda primero en SQLite
  • Sincroniza cuando hay conexión GSM
  • Cero pérdida de eventos mientras el dispositivo está encendido
  • Batería de respaldo: 1h de captura garantizada ante falla de energía

🔐 Seguridad mTLS

  • Cada cámara: certificado X.509 único
  • Autenticación mutua con el broker
  • Revocación remota desde la nube

🔁 Idempotencia

  • Cada evento tiene event_id UUID
  • INSERT ... ON CONFLICT DO NOTHING
  • Duplicados descartados silenciosamente

Patrón Adapter
Un Worker .NET por país. Si una BD cambia su protocolo, solo se modifica ese adaptador — el sistema central no se toca.
Keycloak como broker
El sistema habla solo con Keycloak. Keycloak habla con LDAP, AD, SOAP, REST de cada país. JWT estándar para todos.

Decisiones Clave de Diseño

MQTT sobre HTTP para dispositivos IoT

Conexión persistente, bajo consumo de ancho de banda, QoS integrado. HTTP requiere una conexión por evento — inviable con 1 GB RAM y GSM intermitente.

→ MQTTnet (.NET) · Mosquitto → EMQX a escala

Patrón Adapter por país (no ETL centralizado)

Aísla la complejidad de cada integración. Cambios en una BD externa solo afectan su adaptador. Nuevos países = nuevo Worker Service, sin tocar el core.

→ Un Worker .NET por país · Formato estándar hacia PostgreSQL

Keycloak como broker de identidades

Open source, sin lock-in de proveedor. Habla LDAP, AD, SOAP y REST — el sistema nunca toca los sistemas de autenticación de cada país directamente.

→ Descartados: Azure AD B2C (lock-in), Auth0 (costo), IdentityServer (desarrollo manual)

Dead Letter Queue + monitoreo activo

Mensajes corruptos no bloquean el flujo principal. 5 reintentos con backoff exponencial vía MassTransit, luego DLQ. Grafana alerta si DLQ > 0.

→ UseMessageRetry · Prometheus plugin RabbitMQ · Alert Grafana
